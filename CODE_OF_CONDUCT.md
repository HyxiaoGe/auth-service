# 贡献者公约行为准则

## 我们的承诺

我们作为成员、贡献者和维护者，承诺让每个人都能在不受骚扰的环境中参与本项目，不因年龄、体型、可见或不可见的残障、族裔、性别特征、性别认同与表达、经验水平、教育程度、社会经济地位、国籍、个人外貌、种族、宗教或性取向而受到区别对待。

我们承诺以开放、友善、多元、包容且健康的方式行动和交流。

## 行为标准

有助于营造积极环境的行为包括：

- 对他人表现出同理心和善意；
- 尊重不同的观点、经验和技术选择；
- 友善地给予和接受建设性反馈；
- 承担责任并向受到影响的人道歉，努力从经历中学习；
- 关注对社区整体最有利的结果，而不仅是个人偏好。

不可接受的行为包括：

- 使用带有性暗示的语言或图像，以及不受欢迎的性关注或挑逗；
- 挑衅、侮辱或贬损性评论，以及人身或政治攻击；
- 公开或私下骚扰；
- 未经明确许可发布他人的私人信息；
- 在专业协作环境中可合理视为不当的其他行为。

## 执行责任

维护者负责澄清和执行可接受行为标准，并会对不当、威胁、冒犯或有害行为采取适当且公平的纠正措施。

维护者有权删除、编辑或拒绝不符合本行为准则的评论、提交、代码、Wiki 编辑、Issue 和其他贡献，并会在适当时说明处理理由。

## 适用范围

本行为准则适用于所有项目空间，也适用于个人在公共空间代表本项目或社区的情况。代表项目的情形包括使用官方账号、通过官方渠道发布内容，或作为项目指定代表参加线上或线下活动。

## 报告与执行

如遇滥用、骚扰或其他不可接受行为，请通过仓库所有者的 GitHub 个人资料中公开的联系方式，请求与维护者私下沟通。不要在公开 Issue 中披露受害者身份或其他敏感细节。发生在 GitHub 平台上的严重滥用也可通过 [GitHub Report Abuse](https://support.github.com/contact/report-abuse) 报告。

所有投诉都会被及时、公平地审查。维护者有义务尊重报告者的隐私与安全。

## 执行指引

维护者将根据行为的性质、影响和持续性采取以下措施：

1. **纠正**：私下说明行为不当及其影响，并要求立即纠正。
2. **警告**：对单次事件作出正式警告；若继续相关行为，将采取进一步措施。
3. **临时限制**：在限定时间内禁止参与项目互动或与相关人员公开交流。
4. **永久限制**：对持续违反准则、骚扰个人或表现出攻击性模式的参与者实施永久限制。

## 归属

本行为准则改编自 [Contributor Covenant 2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct.html)，其社区影响指南亦可在该网站查阅。
